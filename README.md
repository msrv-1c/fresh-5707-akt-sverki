# Номер и дата в акте сверки для 1С:Бухгалтерии

Демо-проект [Конвейера 1С](https://gitsell.ru/1c): небольшая идея из практики 1С превращена в готовое расширение с открытым исходным кодом.

Расширение добавляет в печатную форму **акта сверки взаиморасчетов** строку с номером, датой и контрагентом документа. Это удобно, когда акт сохраняют в PDF или отправляют контрагенту и нужно сразу видеть, к какому документу относится печатная форма.

## Зачем Это Нужно

В типовом процессе акт сверки часто печатают или выгружают из документа 1С. Если в печатной форме не хватает реквизитов самого документа, пользователю приходится сверять их отдельно или переименовывать файл вручную.

Расширение дополняет печатный результат: в начале табличного документа появляется строка вида **"Акт сверки № ... от ..."**.

## Что Получает Пользователь

- Автоматическое добавление номера и даты в печатную форму акта сверки.
- Вывод контрагента рядом с реквизитами документа, если он заполнен.
- Сохранение привычного типового сценария печати.
- Доработку через расширение, без изменения основной конфигурации.
- Исходники, документацию и воспроизводимую сборку CFE.

## Что Показывает Этот Репозиторий

Это пример аккуратного перехвата стандартной печати:

- обработчик расширения выполняется после типового формирования печати;
- доработка применяется только к макету `АктСверки`;
- данные документа читаются точечно по ссылке первого печатаемого объекта;
- результат добавляется в уже сформированный табличный документ.

## Как Это Выглядит В 1С

Пользователь печатает акт сверки как обычно. В сформированной печатной форме появляется строка:

```text
Акт сверки № <номер> от <дата>, <контрагент>
```

## Проверка

Расширение рассчитано на работу в **1С:Бухгалтерии предприятия 3.0**.

Проверяемые сценарии:

- печать акта сверки выполняется штатной командой;
- в печатной форме появляется строка с номером и датой документа;
- доработка не затрагивает другие печатные формы;
- расширение не использует файловую систему, COM, внешние HTTP-вызовы или привилегированный режим.

Техническое описание вынесено в [docs/IMPLEMENTATION.md](docs/IMPLEMENTATION.md).

## Сборка CFE

Готовый файл расширения после сборки создается в `dist/ГСК_АктСверкиНомер.cfe`.

Для быстрой пересборки из XML-исходников используется Python-скрипт:

```powershell
python scripts/build_cfe.py --file "<путь к тестовой базе>" --user "<пользователь>"
```

Также можно использовать серверную базу:

```powershell
python scripts/build_cfe.py --server "<сервер 1С>" --ref "<имя базы>" --user "<пользователь>"
```

Скрипт загружает исходники из `src` в расширение и выгружает свежий `.cfe` в каталог `dist`.

## Для Кого Этот Пример

Для бухгалтеров, которые регулярно отправляют акты сверки контрагентам.

Для разработчиков 1С, которым нужен пример доработки печатного результата через расширение.
